About

Details

My biggest interests are in the areas of Distributed Computing and Machine Learning; the latter of which is backed up with graduate-level coursework in neural networks. I see them both playing key roles in the acceleration of scientific discovery and optimization of currently inefficient processes, in addition to just being fascinating in theory.

My work shows a general trend in systems programming and Deep Learning, but I think I can make the most strides in quickening their adoption by novices and professionals alike. As an introvert raised to be an extrovert, a backend heavy programmer interested by UI/UX, I straddle lines. I'd like to one day use my emerging leadership capabilities and knack for design to make these enigmatic yet powerful technologies more accessible to those who could use them most.

Why Computer Science?

I once heard a saying that folks go into the computing field for two main reasons: A) Comfort, and/or B), Impact. While it's up to the individual to decide which is more important, my journey to computer science has been a curved one pulled by the latter -- A longing to make this home of ours a little bit brighter, however indirect it may be.

So I tried my hand out. I was an agent for Search and Rescue in Seattle for many years, and had the privilege of leading non-profits. Although I strongly considered a career in medicine -- as I saw it to be the best way to serve others--, I also took a step back to consider a less traditional option: computing.

It soon dawned on me just how much influence a single person could have in technology. Instead of making diagnoses personally, I could help make a diagonstic tool that might aid countless doctors. What if instead of becoming an environmental engineer, I improved routing algorithms to save internet users time, and thus energy? Could I indirectly help labs the world over by increasing the speed and experience of scientific computation? It was over before it began. An interest in the What, How, and Why of this discipline with yet so much untapped potential took hold.

High Throughput Autonomous Proteomic Pipeline Salomon Research Group

Previous Next

Project Description

Built a parallel, high throughput data pipeline for the Salomon Research Group, a lab dedicated to researching proteins in human T-cells to better understand cell signaling and immune responses. The goal was to implement a new searching algorithm described here that could significantly improve retention of data when comparing peptides that we find in our data to a known database (in this case, known homo sapiens proteins). The pipeline took information directly from lab instruments, queued up samples, performed searching on the new search engine, extracted relevant information, loaded it into a relational database, and performed statisical analysis.

Project Details

  • Series of programs (AutoUpdate, AutoSearch, AutoLoad, in Java) executed in parallel across multiple computers, communicating using the Java File Transfer system.
  • Increased data yield by 10-40%, depending on sample.
  • Now allows the lab to use state of the art Tandem Mass Tag data (TMT), in addition to existing experiment types

XML Search Engine with Pagerank CS 180 (Spring 2017)

Previous Next

Project Description

Built a search engine in Scala that parses a corpus of XML documents (in this case, wikipedia pages). Uses the concepts of term frequency, inverse document frequency, and Google's Pagerank algorithm to return up to 10 of the most relevant docs based on a user query from a REPL.

Project Details

  • To use, download from Github and run the parser program from your terminal. By default, a medium sized corpus is used (~10000 wikipedia pages in XML).
  • Run with a command like |Insert here |. The "p" is for Pagerank usage, optionally.
  • This program was created for a Computer Science class at Brown University. We were given support code to stop and stem words in the corpus in order to standardize them, as well as some beginning structural code for building Pagerank.

Bash Command Line Implementation CS 330 -- Computer Systems (Fall 2017)

Previous Next

Project Description

Used C to create a working version of the bash command line for a course on Computer Architecture/Systems. When downloaded, you can run it by first making an exectuable with the Makefile (type "make 33sh" when in the working directory), followed by ./33sh. Then try it out! Note that right now, it does not show the path of the current working directory. The cd command and navigation still work, however.

What it can do

  • Handle error prone input at all levels
  • Run any specified programs and its arguments successfully
  • Handle builtin commands (cd, lm, rn, exit, fg, bg)
  • Keep track of multiple jobs (either stopped or in the background)
  • Reap jobs
  • Perform file redirection
  • Move jobs to foreground or background
  • Handle signals (either from keyboard or the system).

Sequence to Sequence Machine Translation Model CS 1470 -- Deep Learning (Fall 2017)

Previous Next

Project Description

Used TensorFlow and the idea of Sequence to Sequence Machine Translation to translate from French sentences to English sentences. The idea is that we can feed windows of sentences as training material instead of individual words, using the scope provided by a Recurrent Neural Network (I used a GRU in particular to acheive this type of memory).

Project Details

  • Currently trains off of ~1 million french sentences. A smaller test set is then used to see how the model performs, by taking the output of the network and comparing it to what it should have been
  • Word per word accuracy is roughly ~70%
  • Unfortunately can only take in a corpus at the moment, so it's not interactive. Feel free to peruse the architecture, however.
